NOSQL:

概述:

​ NoSQL(NoSQL = Not Only SQL)，意即“不仅仅是SQL”，是一项全新的数据库理念，泛指非关系型的数据库。

​ 关系型数据库: mysql(DBMS)

​ 数据存储时,数据和数据之间存在着关联关系

​ 关系型数据库中的数据存放在硬盘上.

​ 数据库管理系统: DBMS

​ 数据库: database

​ 表: table

​ 非关系型的数据库:

​ 数据存储时,数据和数据之间不存在关联关系,类似于map

​ 非关系型数据库中的数据存放内存中.

​ 非关系型数据库的 吞吐量 是海量的

宕机: 非正常停止服务

NOSQL作用:

​ 解决三高问题:

​ 高并发: 同时发生的事情

​ 高负载(高存储): 可以存放海量的数据

​ 高可用(高扩展):

Redis:

概述:

​ Redis是用C语言开发的一个开源的高性能键值对（key-value）数据库

数据结构:

​ 字符串类型 string
​ 散列类型 hash 类似于map
​ 列表类型 list 有序可重复
​ 集合类型 set 无序不可重复
​ 有序集合类型 sortedset 有序不可重复

redis应用场景:

​ 做缓存: 缓存的是使用频次较高,但不是特别重要的数据.

​ 请求来时,优先查询非关系型数据库,

​ 如果没有查到则查询关系型数据库,从关系型数据库中查询到结果后,将结果存放到非关系型数据库中,并将结果返回给浏览器.

​ 如果查询到了,直接将查询结果返回给浏览器即可

​ 当用户执行 增 删 改操作时,优先操作关系型数据库, (数据丢失)

​ 操作完毕后,删除非关系型数据库中的相关数据. (数据同步)

redis安装:

​ 解压缩即可

redis启动和关闭:

​ 启动:

​ 方式1: 了解

​ 启动服务器: 双击 redis-server.exe 文件

​ 启动客户端: 双击 redis-cli.exe 文件

​ 方式2: 使用指定配置文件开启服务 (会持久化)

​ 启动服务器: 在dos命令中输入 redis-server.exe redis.windows.conf

​ 启动客户端: 在dos命令中输入 redis-cli.exe

​ 关闭:

​ 方式1: 点击 X 号 了解

​ 方式2: 正常关闭

​ 在dos命令中输入 redis-cli.exe shutdown

redis命令:

redis持久化:

​ 注意: 要想使用redis的持久化操作,启动redis时必须采用配置文件.

一种是RDB方式: (默认)

​ rdb方法在持久化数据时,采用 快照机制.

​ 执行快照的时机:

​ save 900 1 15分钟

​ save 300 10 5分钟

​ save 60 10000 1分钟

​ 特性:

​ 保存的是最终的结果数据

​ RDB方式可能导致数据丢失

​ 执行效率高

​ 启动服务器时,采用配置文件的方式启动.

​ redis服务器正常关闭时,会采用快照机制持久化数据.

一种是AOF方式:

​ 使用AOF方式持久化时,需要在配置文件中开启AOF持久化操作

​ 将 appendonly 的值改为yes

​ 特征:

​ 持久化的是每次执行的命令,不会丢失数据,但性能较低
