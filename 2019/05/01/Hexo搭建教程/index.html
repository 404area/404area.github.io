前言

​ 首先Hexo的搭建本身并没有什么技术含量,唯一要说的地方就是如果新手搭建的话可能会非常的”困难”,我为什么这样说呢?因为它涉及到了较多的命令行以及非常繁琐的配置文件的更改.甚至如果想要更多个性化设置就需要修改css等文件.对代码的基础功底还是有一些考验的.笔者刚刚接触就发现网上的教程多而不准,几乎个个有坑,完全不是新手教程的教程.有感于此,所以就写一篇关于Hexo搭建的教程来帮助那些希望走进博客世界大门的新手来学习.由于笔者水平有限,有误之处望不吝赐教.

介绍

​ Hexo 是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。而本文接受的就是使用Hexo+GitHub+个人域名搭建个人博客.使用这样的组合有什么优点呢?当然是非常多的,我感觉首先是开源+免费+轻量级+全静态! 这是我选择的重要原因.并且本教程基于官方的文档并进行了丰富.

​ 在这里我还想补充一句:我们生活在一个神奇的地方,这个地方被叫做”天朝”,这个地方是有一个具有”与世隔绝”的生态环境.在我们工作的互联网领域亦是如此.那么就对于我们从事互联网领域的工作者产生了一些工作上的困扰.我们自己购买的域名,主机还需要通过一种国家机器的颁发的许可才可以联通这个”世界”.说到这里,有经验的可能就会知道这是一种叫做”备案”的措施.凡是运行在”天朝的互联网”都必须经过这道手续.我想这是基于国家安全上的考虑实施的这些政策.但笔者认为这对于互联网的初衷:自由+平等+分享,有些”背道而驰”原谅我这个词可能用的不是那么准确.但笔者想表达的是本人也非常的厌恶此举,尤其是在信息泄露非常严重的当今社会.所以我选择了通过在github上进行搭建.而不是通过购买主机域名进行备案搭建.

安装

安装前提

安装 Hexo 相当简单。然而在安装前，您必须检查电脑中是否已安装下列应用程序：

  • Node.js (Should be at least nodejs 6.9)
  • Git

如果您的电脑中已经安装上述必备程序，那么恭喜您！接下来只需要使用 npm 即可完成 Hexo 的安装。

命令介绍

init

新建一个网站。如果没有设置 folder ，Hexo 默认在目前的文件夹建立网站。

new

新建一篇文章。如果没有设置 layout 的话，默认使用 _config.yml 中的 default_layout 参数代替。如果标题包含空格的话，请使用引号括起来。

generate

生成静态文件。

选项 描述
-d, --deploy 文件生成后立即部署网站
-w, --watch 监视文件变动

该命令可以简写为

publish

发表草稿。

server

启动服务器。默认情况下，访问网址为： http://localhost:4000/。

选项 描述
-p, --port 重设端口
-s, --static 只使用静态文件
-l, --log 启动日记记录，使用覆盖记录格式
deploy

部署网站。

参数 描述
-g, --generate 部署之前预先生成静态文件

该命令可以简写为：

render

渲染文件。

参数 描述
-o, --output 设置输出路径
migrate

从其他博客系统 迁移内容。

clean

清除缓存文件 (db.json) 和已生成的静态文件 (public)。

在某些情况（尤其是更换主题后），如果发现您对站点的更改无论如何也不生效，您可能需要运行该命令。

list

列出网站资料。

version

显示 Hexo 版本。

选项
安全模式

在安全模式下，不会载入插件和脚本。当您在安装新插件遭遇问题时，可以尝试以安全模式重新执行。

调试模式

在终端中显示调试信息并记录到 debug.log。当您碰到问题时，可以尝试用调试模式重新执行一次，并 提交调试信息到 GitHub。

简洁模式

隐藏终端信息。

自定义配置文件的路径

自定义配置文件的路径，执行后将不再使用 _config.yml。

显示草稿

显示 source/_drafts 文件夹中的草稿文章。

文件结构介绍

  • node_modules: 依赖包
  • public：存放生成的页面
  • scaffolds：生成文章的一些模板
  • source：用来存放你的文章
  • themes：主题
  • _config.yml: 博客站点配置文件
